Why BMA?

There is a major shift towards scale-out deployments in datacenters. Scale-out is driven by a new era of software (On-Prem Cloud, HPC, AI/ML, Containers, VMWare Virtualization, VDI, etc.). However, organizations are facing significant friction with the onboarding, provisioning and management of their physical hardware. Hardware is an expensive and difficult resource to deploy within a data center driving organizations to move to the cloud.

RedefinIT BMA addresses the key challenges in managing bare-metal servers by providing a simplified approach to OS installation and Firmware updates. It uses modern industry standard technologies and protocols to automate OS installation and Firmware updates in a secure, faster and reliable way. This product supports multiple vendor hardware and multiple operating systems and hypervisors, making it a versatile tool in the modern enterprise datacenter. It is highly scalable to handle large-scale deployments. With RESTful API interface, this tool can be easily integrated with O&M (orchestration and management) tools such as Ansible, Terraform etc. for complete end-to-end automation of Day-1 and Day-2 operations for bare-metal server infrastructure.

A Simplified Approach

Key Benefits of BMA:

RedefinIT BMA reduces the complexity in bare-metal server deployment by providing a simplified approach to automate the OS installation and Firmware update process. BMA significantly reduces the cost as well as the time consumption of both processes. Some of the notable advantages are as follows:

  • Deploys bare-metal servers with OS faster.
  • Combines f/w update and OS installation into single task to one board new servers within minutes
  • Bulk deployment with ease and maximum parallelization to reduce time to bring up the environment.
  • Single tool for provisioning multiple hardware vendors. Current version supports HPE and Dell servers.
  • Single tool for deploying multiple operating systems and hypervisors. Current version supports RHEL, VMWare ESXi. More operating systems support in the roadmap.
  • The deployment and provisioning performed using OOB (Out Of Band management network) networking to avoid any interference with production environment.
  • Easily integrates BMA with O&M (Orchestration and management) tools such as Ansible, Terraform using the simple REST API servers.
  • Eliminates dependency on security vulnerable PXE, IPMI, DHCP and rely on open standards and secure alternatives such as Redfish, SSL encrypted communication (HTTPS). Uses HTTPS to communicate with the server BMC.
  • Reduces pre-deployment preparation efforts by directly accepting OS installation media as-is from OS vendors.

Features

RedefinIT BMA uses modern industry standard technologies such as DMTF Redfish, Virtual Media, UEFI, HTTPS/TLS for faster, reliable and secure bare-metal server provisioning. This product not only supports multi-vendor server hardware and operating systems but also hypervisors, making it a versatile tool in the modern enterprise datacenter. Some of the significant features are as follows:

  • API-driven OS provisioning tool using secure REST API interface.
  • Uses modern industry standard approach using Redfish (instead of IPMI) and BMC’s Virtual Media feature instead of PXE for presenting the OS media to the servers.
  • Hardware discovery using Redfish API to BMC instead of OS based hardware discovery to speed up the hardware discovery process.
  • Uses kickstart for achieving unattended OS installation.
  • Configures local storage drives using Redfish API.
  • Option to perform Firmware update abstracting hardware vendor-specific update tools.
  • Built for bare-metal provisioning as against supporting Virtual Machines too.
  • JSON format data input.
  • Supports multiple hardware vendors.
  • Supports operating systems including CentOS, RHEL, SUSE, VMWare ESXi, and Windows.
  • Supports multi-vendor hardware that is DMTF Redfish® compliant (HPE, Dell to start).
  • Supports multiple operating systems (ESXi, RHEL, CentOS, Ubuntu, SLES, Windows).
  • Static IP assignment support.
  • Atomic installation of OS (eliminating need for first boot operations).
  • Installs and upgrades Firmware.
  • Injecting OS drivers into the OS media before the installation.
  • Supports bulk deployment.
  • JSON data format.

Use Cases

VCF Automation

VCF deployment needs the physical hosts installed with the right version of ESXi hypervisor; which is installed with an IP address from the management network assigned to one of its network ports. The VCF software can automate the rest of the steps in creating VCF environment with VCenter, VSAN and NSX, ready for production.

VSAN needs local drives in a clean state without any filesystem partitions as it can be tedious to cleanup the drives before deploying the servers to VCF. If the initial physical hosts deployment can be automated, then end-to-end automation can be achieved for VCF Day-1 and Day-2 operations.

BMA can automate bare-metal physical hosts deployment with the following:

  1. Updates the server Firmware using Firmware updates bundle in ISO format.
  2. Installs OS with the following customizations:
    1. Assigns management IP address to one NIC.
    2. Cleanup of all local drives by removing any pre-existing filesystem partitions.
    3. Enables/disables services as mandated by VCF pre-requisites.

Kubernetes Containers Platformation

With the increasing adoption of container platforms for running enterprise applications, there is an opportunity to eliminate the licensing costs and performance costs from virtualization. The containers can give the isolation and multi-tenancy to run multiple workloads on a single host which has so far been possible only with virtualization. However, most container platform deployments are based on virtualization infrastructure as against physical infrastructure.This is in part due to hesitancy to run containers on bare-metal servers without adding a virtualization layer in between the physical infrastructure and the container platform. The primary reason is the complexity involved with managing the bare-metal servers, which gets worse with managing multi-vendor hardware. The challenges with bare-metal servers does not exist in the virtualization as a VM is the same irrespective of the underlying physical infrastructure. The tooling for virtualized environment can be totally independent of the vendor of the underlying physical hardware. However, this is not the case with multi-vendor bare-metal infrastructure.There will be significant cost saving and increased utilization of compute resources when the container platforms run on bare-metal servers. This needs tooling that allow the deployment and provisioning to be fully automated with multi-vendor hardware support. The minimum the automation code to handle different hardware (with different model/generation/vendor), the better the automation experience.
